      • Linux 使用ldap 作为认证服务器

      Post on 2018-01-11

      顾名思义，Linux作为客户端 ，认证服务器为 ldap

      安装

      yum install openldap-clients nss-pam-ldapd nscd
      卸载 sssd
      yum remove sssd -y

      配置客户端

      authconfig --enableldap --enableldapauth --ldapserver="hadoop" --ldapbasedn="dc=chinainvs,dc=com" --disableldaptls --update

      /etc/pam.d/system-auth 的配置,加入了pam_ldap.so的模块，主要用于系统本身的一些认证如：su - user

      echo “session optional pam_mkhomedir.so skel=/etc/skel umask=0022” >> /etc/pam.d/system-auth
      sed -i ‘s#pam_sss#pam_ldap#g’ /etc/pam.d/system-auth

      /etc/nsswitch.conf 的配置，配置认证顺序，比如认证时先走file还是ldap还是nis

      sed -i ‘s#sss#ldap#g’ /etc/nsswitch.conf ## services 不需要改，还是sss

      /etc/openldap/ldap.conf的配置，主要用于查询OpenLDAP服务器所有条目信息

      /etc/pam_ldap.conf 的配置，配置c/s交互中走pam的认证

      /etc/nslcd.conf 的配置,配置客户端与后ldap认证服务器进行交互，

      /etc/sysconfig/authconfig 的配置文件

      sed -i ‘s#USESHADOW=no#USESHADOW=yes#g’ /etc/sysconfig/authconfig
      sed -i ‘s#USELDAPAUTH=no#USELDAPAUTH=yes#g’ /etc/sysconfig/authconfig
      sed -i ‘s#USELOCAUTHORIZE=no#USELOCAUTHORIZE=yes#g’ /etc/sysconfig/authconfig
      sed -i ‘s#USELDAP=no#USELDAP=yes#g’ /etc/sysconfig/authconfig
      sed -i ‘s#FORCELEGACY=no#FORCELEGACY=yes#g’ /etc/sysconfig/authconfig

      /etc/pam.d/password-auth 的配置，各种密码的认证走的一个配置文件，其中sshd会in clude这个模块，所以sshd认证的相关参数配置在此

      sed -i 's#sss#ldap#g' /etc/pam.d/password-auth ## services 不需要改，还是sss
      /etc/ini.d/sshd restart
