安装ES

配置
  • 不依赖zookeeper,有自己的分布式协调机制
  • 在$ES_HOME/config/elasticsearch.yml配置：
    cluster.name同一个集群必须相同，nodename不能相同，一般是hostname；
    要配置network.host才能别的机器或者网卡访问，否则只能是127.0.0.1或者localhost访问
  • yaml 文件的 配置项一定要顶格写，要不然报错,:后面要有一个空格。
  • /bin/elasticsearch 不能用root启动，因为之前出过事,改为其他用户

es 默认端口：9200,kibana 默认端口:5601

安装Kibana

  • 在es集群的任意一台机器上安装
  • Kibana安装目录中，config/kibana.yml:
    elasticsearch.url:http://IP:9200,该IP可以是es集群中的任意一台。

  • 安装Marve插件 - es的集群监控工具

  • POST 和 PUT 的区别
    : post 一般用于新增，作用于资源集合
    : put 一般用于修改，作用于单个资源
