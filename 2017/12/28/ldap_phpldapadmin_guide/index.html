      • rh6.8 环境搭建 ldap + phpldapadmin

      Post on 2017-12-28

      前置条件 rh6.8 iso 挂载作为本地yum源

      安装OpenLDAP

      yum install openldap openldap-* -y

      查看：rpm -qa | grep openldap
      可以看到

      配置ldap，包括DB_CONFIG 和 slapd.conf

      生成 ldap 管理员密码,下面会用到

      slappasswd -s ldap >> /root/ldap_pw

      修改slapd.conf，主要配置dc和rootpw

      sed -i 's#dc=my-domain#dc=china#g' /etc/openldap/slapd.conf
      可以看到 suffix rootdn 都改成了想要的dc了。
      添加rootpw
      rootpw {SSHA}hmFq6su/1aUnM954IYc8chaWcy5lLOXd

      检测并重新生成ldap数据库

      可能会报错，说failed的，不知道你们有没有碰到，反正我是碰到了。别管它，不影响。

      修改ldap相关文件的权限

      启动sldap服务

      /etc/init.d/sldap start
      查看进程
      ps aux | grep ldap

      安装migrationtools

      yum install migrationtools -y

      ###修改 /usr/share/migrationtools/migrate_common.ph 配置

      生成 base.ldif 文件

      /usr/share/migrationtools/migrate_base.pl >base.ldif

      添加base.ldif 到 ldap

      ldapadd -x -D "cn=Manager,dc=china,dc=com" -W -f ./base.ldif
      提示输入管理员密码。

      查询一下，看是否加入成功

      ldapsearch -x -D "cn=Manager,dc=china,dc=com" -b "ou=Aliases,dc=china,dc=com" -W

      安装phpldapadmin

      yum install httpd -y
      因为没有配置phpldapadmin的yum源，我们下载下来官方地址，本地安装,
      yum localinstall -y phpldapadmin-1.2.3-10.el6.noarch.rpm

      配置etc/httpd/conf.d/phpldapadmin.conf

      修改/etc/phpldapadmin/config.php

      启动httpd，打开phpldapadmin 并登陆

      在浏览器输入:
      hadoop/phpldapadmin
      登录时，登录DN输入：cn=Manager,dc=china,dc=com
      密码就是当时设置的密码 , 如果是管理员需要操作，请 不要勾选anonymous
